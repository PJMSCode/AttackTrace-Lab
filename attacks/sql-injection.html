Attack Vector Lab

SQL Injection Attempt

This lab demonstrates how suspicious database query manipulation attempts can be represented as structured security telemetry and reviewed by detection logic.

View Simulated Endpoint View JSON Log

Overview

What this attack is trying to do

Attack Goal

SQL injection attempts try to manipulate database queries by placing unexpected SQL syntax inside user-controlled input fields.

Target Surface

Search bars, login forms, filters, and profile lookup pages are common places where attackers may test injection payloads.

Detection Value

Detecting these patterns helps security teams identify probing activity against public-facing web applications.

Payload Examples

Suspicious patterns this lab tracks

High ' OR '1'='1

Classic tautology pattern used to alter query logic.

Medium admin'--

Comment operator attempt that may be used to ignore the rest of a query.

High ' UNION SELECT username, password FROM users--

Union-based pattern attempting to retrieve unauthorized database fields.

Detection Logic

How the rule should think

Check Input

Inspect user-controlled fields such as search queries, login usernames, filters, and profile IDs.

Match Patterns

Look for suspicious SQL operators, tautologies, comment markers, and union-select style payloads.

Create Alert

Return a structured alert with severity, route, payload, source IP, and analyst-friendly reasoning.
