Detection Engineering Portfolio Project

Simulated Web Attacks. Structured Security Logs. Custom Detection Rules.

LONAVA is a security engineering project that simulates common web application attack attempts, records each event as JSON telemetry, and analyzes the activity with Python-based detection logic.

View Sample Log See Detection Flow

Project Overview

Built to demonstrate practical security engineering skills

Attack Simulation

Models common web attack attempts inspired by training labs such as SQL injection, XSS, brute force login attempts, and path traversal.

Security Telemetry

Each attack attempt is converted into structured JSON logs containing timestamps, routes, payloads, source IPs, severity, and MITRE mapping.

Detection Logic

Python detection rules inspect the generated logs and produce analyst-friendly alerts for suspicious behavior.

Attack Coverage

Initial attack types tracked by the lab

High

SQL Injection Attempt

Detects suspicious input patterns such as tautologies, comment operators, and payloads targeting database queries.

Medium

Cross-Site Scripting Attempt

Identifies script tags, event handlers, encoded payloads, and suspicious HTML injection attempts.

High

Brute Force Login

Flags repeated failed login attempts from the same source IP or username within a short time window.

Medium

Path Traversal

Detects attempts to access unauthorized files using traversal patterns such as ../ or encoded directory sequences.

Detection Pipeline

How LONAVA turns events into alerts

1

Simulate Request

A user submits input to a vulnerable demo route.

2

Generate Log

The request is normalized into a structured JSON event.

3

Run Rule Engine

Python detection rules inspect the event for suspicious behavior.

4

Create Alert

The detection engine outputs a clear alert with severity and reasoning.
